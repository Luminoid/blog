Using static site generator with GitHub Pages

Posted on 2017-08-27 Edited on 2019-02-05 In Web , Static Site Generator
Symbols count in article: 1.3k Reading time ≈ 1 mins.

There are three ways to deploy static site on GitHub Pages as follows. I tend to use the third scheme myself. The related GitHub documentation can be found here.

Using gh-pages branch

Simply deploy website content on gh-pages branch and maintain code locally.

Advantage

This is the easiest way because many static site generators have build-in deploy command. Users would only need to setup the deployment configuration once and run deploy command every time after the modification. An npm package gh-pages helps doing this, if there is no built-in deploy command.

Disadvantage

Only website content is under the version control. The code that generates the website content is maintained locally.

Let & Const

Block-scoped binding constructs.

let var1 [= value1];
const name1 = value1;

Destructuring

// Array matching
let [a, ,c] = [1,2,3];
a === 1;
c === 3;

// Object matching
let { a, b } = { a: 1, b: 2 };
a === 1;
b === 2;

// Default arguments
function add({a, b, c = 3, d = 4}) {
  return a + b + c + d;
}
add({a:1, b:2}) === 10;

// Swap values
let x = 1;
let y = 2;
[x, y] = [y, x];

// Multiple return values
